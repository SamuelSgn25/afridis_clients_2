📝 NOTES IMPORTANTES
================================================================================

- Le backend utilise PostgreSQL avec l'extension uuid-ossp pour les UUID
- Les mots de passe sont hashés avec bcrypt (10 rounds)
- Les tokens JWT expirent après 15 minutes (access) et 7 jours (refresh)
- Le refresh token est stocké dans un cookie httpOnly sécurisé
- Le logout révoque le token en l'ajoutant à la blacklist
- Les routes protégées nécessitent le header Authorization: Bearer <token>
- Le frontend gère automatiquement le refresh token en cas d'expiration
- Les rôles sont: super_admin, admin, commercial (par défaut)
- Certaines routes (création/modification services) nécessitent le rôle admin
